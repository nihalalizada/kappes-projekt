ARP Cache Poisoning Attack

Angriffsbeschreibung


Das Address Resolution Protocol (ARP) wird zur Identifikation von Geräten innerhalb eines Netzwerk genutzt.
Der Angriff bewirkt ein unbemerktes Eindringen in die Kommunikation zwischen einem Host und dem Gateway. Ein Python Skript sendet kontinuierlich gefälschte ARP Pakete an ein Opfer und das Gateway und flutet so den ARP Cache Table. Es resultiert eine Verletzung des IT Security Ziels Integrity. In Verbindung mit einem DNS Spoof werden vertrauliche Informationen (hier: Kontodaten) mitgelesen und unauthorisiert gespeichert.

Sicherheitsmaßnahmen


https only mode im Browser aktivieren
statischen Eintrag in ARP Table kreieren: sudo arp -s ip address mac address

Voraussetzungen


- Docker
- Python3 mit Scapy Library
- IP Forwarding aktiviert (Mac OS: sudo sysctl -w net.inet.ip.forwarding=1)
- Ettercap installiert


Ablauf


- Webapp starten: docker-compose up -d
- Run arp.py, IP Adressen von Victim und Gateway angeben
- Ettercap Konfiguration:
- etter.conf: Aktivierung von redir_commands. (Zwei für IPv4, zwei für IPv6)
- etter.dns: Festlegen der Redirect Adressen für DNS Spoofing
- Ettercap starten: sudo ettercap -G
- Target hinzufügen
- DNS Spoof Plugin aktivieren
